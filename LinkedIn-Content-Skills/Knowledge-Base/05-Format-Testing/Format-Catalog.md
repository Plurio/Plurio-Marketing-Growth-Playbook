## Detailed Format Specs

### 1. Carousel / Document Posts

**Specs:**
- Format: PDF upload
- Slide size: 1080x1350px (vertical, 4:5) — best for mobile
- Slides: 5-10 optimal
- File size: Max 300MB
- First slide = hook (counts as "thumbnail")

**When to use:**
- Frameworks and models (e.g., "5-step verification chain")
- Step-by-step guides
- Data visualizations
- Checklists and cheat sheets

**When NOT to use:**
- Generic tips without depth
- Style-over-substance (pretty slides, no insight)
- Content that works better as text

### 2. Multi-Image Posts

**Specs:**
- Up to 9 images per post
- Recommended: 3-5 images
- Size: 1200x1200px or 1080x1350px
- Each image should add independent value

**When to use:**
- Before/after comparisons
- Screenshots of real work (dashboards, agent output)
- Data sets with different views
- Team/product visual stories

**When NOT to use:**
- Decorative images without information
- Text-heavy slides (use carousel instead)

**Per-author affinity:**

| Author | Multi-Image Affinity | Best Topics |
|--------|---------------------|-------------|
| Seva | Medium | Before/after dashboards, agent output screenshots |
| Kirill | Medium-High | Agent reasoning screenshots, dashboard comparisons |
| Ella | Medium | Workflow before/after, content system visuals |
| Denis | Medium | Metric before/after (NPS 10→40+, 60→6 managers), system architecture |

### 3. Native Video

**Specs:**
- Format: MP4, MOV
- Aspect ratio: 9:16 vertical (for Video Tab) or 16:9 horizontal
- Duration: <60 seconds optimal (max 10 min)
- Captions: Required (80%+ watch with sound off)
- Upload: Native only (not YouTube/Vimeo links)

**When to use:**
- Product demos ("agent doing X in real time")
- Podcast clips (15-30 second highlights)
- Screen recordings with commentary
- "Day in the life" authenticity

**When NOT to use:**
- Content that needs deep reading (use text/carousel)
- Without captions
- Long videos (>2 min for LinkedIn)

**Per-author affinity:**

| Author | Video Affinity | Best Topics |
|--------|---------------|-------------|
| Seva | Medium-High | Podcast clips, Cursor demos |
| Kirill | Low (untested) | Screen recordings of agent work |
| Ella | Medium | Workflow recordings |
| Denis | Medium (untested) | Conference talk clips (20+ talks available), product demo walkthroughs |

### 4. Text Only

**Specs:**
- Max: 3,000 characters
- Sweet spot: 1,200-1,800 characters (+47% engagement)
- "See more" cutoff: 210-235 characters (60-70% readers lost here)
- First 2 lines = hook (must stop scroll)

**When to use:**
- Hot takes and contrarian positions
- Personal stories and reflections
- Quick insights that don't need visuals
- Series posts (Part 1/4, etc.)

**When NOT to use:**
- Content with lots of data (use carousel)
- Step-by-step processes longer than 5 steps (use carousel)
- Content that benefits from visual proof

**Per-author affinity:**

| Author | Text Affinity | Best Topics |
|--------|-------------|-------------|
| Seva | High | Philosophy, frameworks, series |
| Kirill | High | Provocative takes, personal stories |
| Ella | Medium-High | Process reveals, tension-resolution |
| Denis | High | Contrarian CPO takes, metric transformations, system reveals |

---

### 5. Polls

**Specs:**
- Options: 3 optimal (not 2, not 4)
- Duration: 7 days
- Must be a genuine question (not engagement bait)

**When to use:**
- Only for real audience research
- When you genuinely need community input
- Max 1 per month

**When NOT to use:**
- As a growth tactic (algorithm suppresses)
- Obvious/leading questions
- Frequently

---

### 6. External Links

**Specs:**
- ~60% reach penalty (all sources confirm)
- "Link in first comment" also penalized (2026)
- LinkedIn official: "No penalty if content leads with value" (contradicts data)

**Best practice:**
- Provide full value in the post first
- Add the link after 300+ characters
- Consider: do you really need the link?
- If linking a podcast/article: summarize key points, the link is secondary

## Content Patterns (inside formats)

Patterns are delivery structures that work inside any of the formats above. The same pattern can be wrapped in Text Only, Carousel, or Multi-Image.

### Timeline Pattern (Day 1 → Day 4 → Day 7)

**Structure:**
```
Hook: what's at stake (budget figure / size of mistake / deadline)
Day 1: signal / decision / action
Day 4: what happened / who waited / who didn't
Day 7: result / how much was lost or gained
Lesson: one takeaway with no moralizing
```

**When it works:**
- Real case from a transcript (not invented)
- Numbers at each point of the timeline
- A story with a "turn" — first one decision, then collision with reality
- Conflict of "instinct vs data" or "expectation vs action"

**When it does NOT work:**
- Without numbers / without real stage identification
- If all three points = "everything worked" (no drama)
- For abstract ideas without a concrete project anchor

**Per-author affinity:**

| Author | Timeline Affinity | Best Topics |
| ------ | ----------------- | ----------- |
| Seva | High | Client cases, pilot stories, founder decisions with numbers |
| Kirill | High | Agent deployment timelines, pilot rollouts, decision chains |
| Ella | Medium | Content system rollouts, process changes |
| Denis | High | Product/team transformations (NPS 10→40+, 60→6 managers) |

**Source constraint:** The Timeline pattern requires real data from transcripts. Any numbers must be in the DIP Stories Bank or in `your-transcripts/`. Invented timelines break the Canonical Source Rule.

## Post Length Guide

| Length | Characters | Best For |
|--------|-----------|----------|
| **Micro** | <500 | Quick hot takes, bold statements |
| **Short** | 500-1,000 | One insight + example |
| **Sweet Spot** | 1,200-1,800 | Full framework or story (+47% engagement) |
| **Long** | 1,800-2,500 | Deep dives (only if compelling throughout) |
| **Max** | 2,500-3,000 | Exceptional content only |
